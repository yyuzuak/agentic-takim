Anahtarlar:
  rl:{tool}:min:{epoch_minute}  — dakikalık sayaç

Redis bağlanamazsa → passthrough (True, 0) — graceful degradation.
"""

## 4. `handle()` Revize Akışı

Mevcut satırlar değişmez; araya yeni katmanlar eklenir:

```
 1. idempotency check                        ← mevcut
 2. fault injection                          ← mevcut
 3. permission check                         ← mevcut
 4. dry_run check                            ← YENİ → simüle sonuç + record(dry_run=true), return
 5. schema validation                        ← YENİ → SCHEMA error (non-retryable)
 6. rate limit check                         ← YENİ → RATE_LIMIT + retry_after (retryable)
 7. handler execute (timeout)                ← mevcut
 8. _record_success (atomic)                 ← YENİ → tool_invocations + compensations
 9. _publish_ok                              ← mevcut
```

### `_record()` yerine `_record_success()`

                           dry_run=False, rate_limited=False, schema_errors=None) -> None:

## 6. Plan-time Schema Validation (Control-Plane)

```python
# services/control-plane/app/orchestrator.py
def validate_tool_plan(plan: list[dict], catalog: dict) -> list[str]:
    """POST /tasks yolunda plan sonrası çağrılır.
    
    Runtime validation ile birlikte defense in depth (INV-3).
    """
    errors = []
    for node in plan:
        if node.get("node_kind") == "tool":
            errs = schema_validator.validate(
                node["tool"], node.get("tool_args", {}), catalog)
            errors.extend(f"{node['node_key']}: {e}" for e in errs)
    return errors
```

## Guardrails

### GR-1 — Compensation yalnızca `side_effect=true` tool'lar

`side_effect=false` (check_stock, generate_pdf) → compensation kaydı oluşmaz.
`side_effect=true + compensation=null` (send_whatsapp, drop_table) → kayıt oluşur ama `compensate_fn=null`.

### GR-2 — Rate-limit retry_count arttırmaz

Ayrı `rate_limit_count` sayacı. Bu sayaç max'e ulaşırsa DLQ; `retry_count` dokunulmaz.
